## F. 인증 시도 횟수 제한

| # | 테스트 항목 | 예상 결과 | 통과 |
|---|-----------|-----------|------|
| F-1 | 잘못된 인증코드 1~4회 입력 | 각각 "인증 코드가 올바르지 않습니다" 에러, 재시도 가능 | |
| F-2 | 잘못된 인증코드 5회 입력 | 5회째에 24시간 잠금 설정 | |
| F-3 | 잠금 후 올바른 코드 입력 | "인증 시도 횟수를 초과했습니다" 에러 (올바른 코드여도 거부) | |
| F-4 | 잠금 후 다른 브라우저/기기에서 시도 | 동일하게 거부 (이메일 기준 추적) | |
| F-5 | 잠금 응답에 실패 횟수/남은 횟수 미포함 확인 | 에러 메시지에 횟수 정보 없음 | |
| F-6 | 1시간 윈도우 만료 후 재시도 (잠금 미도달 상태) | 카운트 리셋, 다시 5회 허용 | |
| F-7 | 24시간 잠금 해제 후 재시도 | 정상적으로 인증 가능 | |
| F-8 | 인증 성공 시 시도 기록 삭제 확인 | DB에서 해당 이메일의 attempt 레코드 없음 | |

## 테스트 환경

- **백엔드**: 로컬 Spring Boot (localhost:8080)
- **프론트엔드**: 로컬 React dev server (localhost:3000)
- **DB**: PostgreSQL (로컬 또는 Docker)
- **도구**: 브라우저 + curl/Postman (API 직접 호출 테스트)
