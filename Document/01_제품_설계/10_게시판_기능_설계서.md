# 10. 게시판 기능 설계서

> 최종 수정: 2026-04-21
> 기술 스택: React 19 + Spring Boot 3.2 + PostgreSQL 16 (기존 프로젝트 통합)

---

## 1. 개요

스태프(manager/admin)가 페이지를 작성하고, 비회원/회원이 등급에 따라 열람하는 게시판 시스템.
Confluence 스타일의 사이드바 트리 구조로 페이지를 계층적으로 관리하며,
이미지 위에 마커를 찍어 주석을 달 수 있는 기능을 포함합니다.

---

## 2. 접근 권한

| 등급 | 글 목록 | PUBLIC 열람 | MEMBER 열람 | STAFF 열람 | 글 작성/수정/삭제 |
|------|---------|------------|------------|-----------|----------------|
| 비회원 (guest) | O | O | X | X | X |
| 회원 (user) | O | O | O | X | X |
| 스태프 (manager/admin) | O | O | O | O | O |

- 글 목록은 **모든 등급에서 볼 수 있음** (제목, 작성자 등)
- 권한이 없는 글을 클릭하면 본문 대신 안내 메시지 표시
- **글 작성/수정/삭제는 스태프(manager/admin)만 가능**
- 게시판에서 manager와 admin은 동일 권한 (스태프로 통합)

### 글 공개 범위

| 값 | 최소 열람 등급 | 설명 |
|---|-------------|------|
| `PUBLIC` | 비회원 | 누구나 열람 가능 |
| `MEMBER` | 회원 (user) | 로그인한 회원부터 열람 가능 |
| `STAFF` | 스태프 | 매니저/관리자만 열람 가능 (내부 공지 등) |

## 4. DB 테이블 설계

### 4.1 posts (게시글)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK (IDENTITY) | 게시글 ID |
| author_id | UUID FK → users.id | 작성자 |
| title | VARCHAR(200) | 제목 |
| content_html | TEXT | 본문 (HTML, TipTap 출력) |
| annotated_images_json | TEXT | 주석 이미지 데이터 (JSON) |
| visibility | VARCHAR(20) | `PUBLIC` / `MEMBER` / `STAFF` |
| parent_id | BIGINT FK → posts.id | 상위 게시글 ID (NULL이면 최상위) |
| sort_order | INT | 같은 부모 내 정렬 순서 |
| view_count | INT | 조회수 |
| is_deleted | BOOLEAN | 소프트 삭제 |
| created_at | TIMESTAMP | 작성일 |
| updated_at | TIMESTAMP | 수정일 |

### 4.2 post_images (게시글 이미지)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK (IDENTITY) | 이미지 ID |
| post_id | BIGINT FK → posts.id | 게시글 ID |
| image_url | VARCHAR(500) | 이미지 경로 |
| original_name | VARCHAR(255) | 원본 파일명 |
| file_size | BIGINT | 파일 크기 (bytes) |
| created_at | TIMESTAMP | 업로드일 |

## 5. API 설계

### 5.1 게시글 CRUD

| 메서드 | 엔드포인트 | 인증 | 설명 |
|--------|-----------|------|------|
| GET | `/api/v1/posts` | 불필요 | 게시글 목록 (페이징) |
| GET | `/api/v1/posts/{id}` | 조건부 | 게시글 상세 (등급별 접근 제어) |
| POST | `/api/v1/posts` | 스태프 | 게시글 작성 |
| PUT | `/api/v1/posts/{id}` | 스태프 | 게시글 수정 |
| DELETE | `/api/v1/posts/{id}` | 스태프 | 게시글 삭제 (소프트) |
| PATCH | `/api/v1/posts/{id}/move` | 스태프 | 게시글 이동 (부모/순서 변경) |

### 5.2 이미지 업로드

| 메서드 | 엔드포인트 | 인증 | 설명 |
|--------|-----------|------|------|
| POST | `/api/v1/posts/images` | 스태프 | 이미지 업로드 (multipart, 최대 5MB) |

### 5.3 이동 API 요청 예시

```json
PATCH /api/v1/posts/3/move
{
  "parentId": 1,     // null이면 최상위로 이동
  "sortOrder": 2     // 같은 부모 내 순서
}
```

### 5.4 상세 응답 — 접근 권한 로직

```
요청자 등급 < 글의 visibility 등급
  → 200 OK (제목/작성자만 반환, 본문 제외, restricted: true)

요청자 등급 >= 글의 visibility 등급
  → 200 OK (전체 반환, restricted: false)
```

## 6. 프론트엔드 구조

### 6.1 파일 구조

```
frontend/src/CategoryPages/Board/
├── BoardPage.tsx              # 메인 (사이드바 트리 + 본문)
├── BoardPage.css
├── PostDetailPage.tsx         # 단독 상세 페이지 (직접 URL 접근용)
├── PostDetailPage.css
├── PostEditorPage.tsx         # 글 작성/수정 (TipTap + 이미지 주석)
├── PostEditorPage.css
├── types.ts                   # 타입 정의
├── components/
│   ├── ImageAnnotator.tsx     # 이미지 마커 주석 컴포넌트
│   └── ImageAnnotator.css
└── extensions/
    └── FontSize.ts            # TipTap 폰트 크기 확장
```

### 6.2 라우팅

| 경로 | 컴포넌트 | 설명 |
|------|---------|------|
| `/board` | BoardPage | 사이드바 트리 + 본문 (메인) |
| `/board/:id` | PostDetailPage | 단독 상세 페이지 |
| `/board/write` | PostEditorPage | 새 글 작성 |
| `/board/edit/:id` | PostEditorPage | 글 수정 |

### 6.3 사이드바 트리

```
┌─────────────────────┐
│ 페이지          + ◀  │  ← 헤더 (새 페이지, 접기)
├─────────────────────┤
│ + 페이지 생성         │  ← 스태프만 표시
├─────────────────────┤
│ ▼ 소방 시뮬레이션      │  ← 최상위 (접기/펼치기)
│   ├ BulC 소개         │  ← 하위 1단계
│   ├ ▶ 사용법          │  ← 하위 (접힌 상태)
│   └ FAQ              │
│ ▶ 회사 소개           │  ← 최상위 (접힌 상태)
│ 공지사항       🔒 회원 │  ← 회원 전용 표시
└─────────────────────┘
```

**드래그 앤 드롭:**
- 항목을 **0.5초 길게 누르면** 드래그 모드 진입
- 다른 항목 위쪽/아래쪽에 놓으면 → 같은 레벨로 순서 이동
- 다른 항목 가운데에 놓으면 → 하위 항목으로 이동
- 시각 표시: 위(주황선), 가운데(점선 테두리), 아래(주황선)

### 6.4 에디터 (PostEditorPage)

**툴바 구성:**
```
[14px ▼] [제목1 ▼] | B I S | • 1. | " </> | 🔓
```

| 영역 | 기능 |
|------|------|
| 폰트 크기 드롭박스 | 12px ~ 32px (기본 14px) |
| 제목 드롭박스 | 본문 / 제목1 / 제목2 / 제목3 |
| 텍스트 스타일 | 굵게(B), 기울임(I), 취소선(S) |
| 목록 | 불릿(•), 번호(1.) |
| 블록 | 인용("), 코드(</>) |
| 공개 범위 | 자물쇠 아이콘 (열림=전체공개, 잠금=회원전용) |

**이미지 주석:**
- 에디터 하단 `+ 이미지 추가` 버튼
- 이미지 선택 → ImageAnnotator 컴포넌트 추가
- 이미지 위 클릭 → 번호 마커 생성
- 마커 드래그로 위치 조정
- 마커별 항목명 + 설명 입력
- 여러 이미지 추가 가능

### 6.5 이미지 주석 데이터 구조

```json
[
  {
    "imageUrl": "/uploads/posts/abc.jpg",
    "markers": [
      { "number": 1, "x": 25.5, "y": 40.2, "label": "머리", "description": "잠자리의 머리 부분" },
      { "number": 2, "x": 55.0, "y": 35.0, "label": "가슴", "description": "날개가 붙어있는 부분" }
    ]
  }
]
```

> x, y는 이미지 크기 대비 퍼센트(%)로 저장 → 화면 크기 변해도 위치 유지

### 6.6 등급별 UI 차이

| 요소 | 비회원 | 회원 | 스태프 (manager/admin) |
|------|--------|------|----------------------|
| 글 목록 | 전체 표시 | 전체 표시 | 전체 표시 |
| 잠금 아이콘 | MEMBER/STAFF 글 | STAFF 글 | 없음 |
| 권한 없는 글 클릭 | 안내 메시지 + 로그인 유도 | 안내 메시지 | — |
| 글쓰기 버튼 | 숨김 | 숨김 | 표시 |
| 수정/삭제 | — | — | 모든 글 |
| 드래그 앤 드롭 | — | — | 가능 |

## 7. 파일 저장 전략

### 개발 환경
- 로컬 디렉토리: `backend/uploads/posts/`

### 운영 환경
- Docker volume 또는 S3

### 보안
- 파일명 UUID 변환
- 확장자 화이트리스트 (jpg, jpeg, png, gif, webp)
- 최대 5MB 제한

## 9. 참고사항

- 기존 인증 시스템(JWT HttpOnly 쿠키) 활용
- 기존 RBAC(admin/manager/user) 역할 체계와 통합
- `is_deleted` 소프트 삭제로 데이터 보존
- 게시글 본문은 HTML로 저장 (TipTap 출력)
- 주석 이미지는 JSON으로 `annotated_images_json` 컬럼에 저장
- 같은 페이지 재클릭 시 조회수 증가 방지
- 게시 버튼 클릭 시 화면 잠금 오버레이 (중복 클릭 방지)
