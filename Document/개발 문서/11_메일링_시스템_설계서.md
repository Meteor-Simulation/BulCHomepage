# 11. 메일링 시스템 설계서

## 1. 개요

BulC Homepage의 메일링 시스템은 두 가지 유형으로 구분하여 운영한다.

| 구분 | 광고성 메일 | 정보성 메일 |
|------|-----------|-----------|
| 목적 | 프로모션, 이벤트, 신제품 안내 | 비밀번호 변경, 라이선스 만료, 결제 완료 등 |
| 수신 동의 | 필수 (opt-in) | 불필요 (서비스 운영 목적) |
| 수신 거부 | 필수 (opt-out 링크 포함) | 해당 없음 |
| 표기 의무 | 제목에 `(광고)` 표기, 발신자 정보, 수신거부 방법 | 없음 |
| 발송 시간 | 야간 제한 (21:00~08:00 발송 금지) | 제한 없음 |

---

## 2. 법적 요건 체크리스트

### 2.1 광고성 메일 관련 (정보통신망법)

- [ ] 제목에 `(광고)` 문구 표기
- [ ] 본문에 발신자 상호, 연락처, 주소 명시
- [ ] 수신거부 링크 포함 (로그인 없이 처리 가능해야 함)
- [ ] 수신 동의 시점 및 방법 기록 보관
- [ ] 야간 발송 제한 적용 (21:00~08:00)
- [ ] 수신거부 처리 후 재발송 방지

---

## 3. 시스템 아키텍처

```
┌──────────────────┐
│     Trigger       │  관리자 수동 발송 / 시스템 이벤트 자동 발송
└────────┬─────────┘
         ▼
┌──────────────────┐
│   Mail Queue      │  DB 테이블 (발송 대기열)
│   (mail_queue)    │  상태: PENDING → SENDING → SENT / FAILED
└────────┬─────────┘
         ▼
┌──────────────────┐
│   Mail Sender     │  Spring Scheduler (주기적 폴링)
│   (Graph API)     │  Microsoft Graph API로 실제 발송
└────────┬─────────┘
         ▼
┌──────────────────┐
│   Mail Log        │  발송 결과 기록 (성공/실패/에러 내용)
│   (mail_log)      │
└──────────────────┘
```

### 발송 흐름

1. **정보성 메일**: 시스템 이벤트 발생 → `MailService.send()` → 큐 적재 → 즉시 발송
2. **광고성 메일**: 관리자가 대상/템플릿 선택 → 큐 적재 → 스케줄러가 야간 제한 확인 후 발송

## 4. DB 테이블 설계

### 4.1 mail_template (메일 템플릿)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | 템플릿 ID |
| name | VARCHAR(100) | 템플릿 이름 |
| type | VARCHAR(20) | `PROMOTIONAL` / `INFORMATIONAL` |
| subject | VARCHAR(200) | 메일 제목 (변수 포함 가능) |
| body_html | TEXT | 본문 HTML (변수 포함 가능) |
| variables | VARCHAR(500) | 사용 가능한 변수 목록 (JSON) |
| is_active | BOOLEAN | 활성 여부 |
| created_at | TIMESTAMP | 생성일시 |
| updated_at | TIMESTAMP | 수정일시 |

### 4.2 mail_queue (발송 대기열)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | 큐 ID |
| template_id | BIGINT FK | 템플릿 ID |
| recipient_email | VARCHAR(200) | 수신자 이메일 |
| recipient_name | VARCHAR(100) | 수신자 이름 |
| variables | TEXT | 치환 변수 (JSON) |
| status | VARCHAR(20) | `PENDING` / `SENDING` / `SENT` / `FAILED` |
| scheduled_at | TIMESTAMP | 예약 발송 시간 (NULL이면 즉시) |
| created_at | TIMESTAMP | 생성일시 |

### 4.3 mail_log (발송 이력)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | 로그 ID |
| queue_id | BIGINT FK | 큐 ID |
| status | VARCHAR(20) | `SENT` / `FAILED` |
| error_message | TEXT | 실패 시 에러 메시지 |
| sent_at | TIMESTAMP | 실제 발송 시각 |

### 4.4 mail_subscription (수신 동의 관리)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | ID |
| user_id | BIGINT FK | 사용자 ID |
| promotional_agreed | BOOLEAN | 광고성 메일 수신 동의 |
| agreed_at | TIMESTAMP | 동의 일시 |
| disagreed_at | TIMESTAMP | 거부 일시 |
| updated_at | TIMESTAMP | 최종 변경일시 |

### 4.5 mail_unsubscribe (수신 거부)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | ID |
| token | VARCHAR(100) UNIQUE | 수신거부 고유 토큰 |
| user_id | BIGINT FK | 사용자 ID |
| unsubscribed_at | TIMESTAMP | 거부 처리 시각 |

## 5. 백엔드 컴포넌트 설계

### 5.1 Service 계층

| 클래스 | 역할 |
|--------|------|
| `MailService` | 메일 발송 요청 생성 (큐에 적재) |
| `MailSendScheduler` | 큐를 주기적으로 폴링하여 Graph API로 발송 |
| `MailTemplateService` | 템플릿 CRUD, 변수 치환 렌더링 |
| `MailSubscriptionService` | 수신 동의/거부 관리 |
| `UnsubscribeService` | 토큰 기반 수신거부 처리 |

### 5.2 Controller

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/admin/mail/templates` | GET/POST/PUT/DELETE | 템플릿 관리 (관리자) |
| `/api/admin/mail/send` | POST | 광고성 메일 발송 (관리자) |
| `/api/admin/mail/logs` | GET | 발송 이력 조회 (관리자) |
| `/api/mail/unsubscribe/{token}` | GET | 수신거부 처리 (비인증) |
| `/api/mypage/mail-subscription` | GET/PUT | 수신 동의 설정 (사용자) |

### 5.3 Rate Limiting

Microsoft Graph API 제한사항:
- 분당 발송 제한 존재 (테넌트별 상이)
- 스케줄러에서 throttling 적용 필요
- 실패 시 재시도 로직 (최대 3회, 지수 백오프)

## 6. 프론트엔드 화면

### 6.1 관리자 페이지

- **메일 템플릿 관리**: 템플릿 목록, 작성/수정 (HTML 에디터)
- **메일 발송**: 대상자 선택 (전체/조건 필터) + 템플릿 선택 + 미리보기 + 발송
- **발송 이력**: 발송 내역, 성공/실패 건수, 상세 로그

### 6.2 사용자 페이지 (마이페이지)

- **수신 설정**: 광고성 메일 수신 동의/거부 토글

### 6.3 수신거부 페이지

- 별도 공개 페이지 (로그인 불필요)
- 토큰 기반 자동 처리
- 처리 완료 메시지 표시

## 7. 메일 템플릿 규격

### 7.1 광고성 메일 필수 요소

```html
<!-- 제목 예시 -->
(광고) [BulC] 신규 라이선스 프로모션 안내

<!-- 본문 하단 필수 포함 -->
<footer>
  <p>본 메일은 광고성 정보가 포함되어 있습니다.</p>
  <p>발신자: ㈜OOO | 주소: OOO | 연락처: OOO</p>
  <p>수신을 원하지 않으시면 <a href="{unsubscribeUrl}">수신거부</a>를 클릭하세요.</p>
</footer>
```

### 7.2 사용 가능한 템플릿 변수

| 변수 | 설명 |
|------|------|
| `{userName}` | 수신자 이름 |
| `{userEmail}` | 수신자 이메일 |
| `{licenseName}` | 라이선스 이름 |
| `{expiryDate}` | 만료일 |
| `{unsubscribeUrl}` | 수신거부 링크 |

## 8. 구현 우선순위

### Phase 1 - 정보성 메일 (법적 부담 낮음)

1. DB 테이블 생성 (mail_template, mail_queue, mail_log)
2. MailService + MailSendScheduler 구현
3. 라이선스 만료 알림, 결제 완료 메일 연동

### Phase 2 - 광고성 메일 기반 구축

1. mail_subscription, mail_unsubscribe 테이블 생성
2. 수신 동의 UI (회원가입 + 마이페이지)
3. 수신거부 페이지 구현

### Phase 3 - 광고성 메일 발송

1. 관리자 템플릿 관리 페이지
2. 대상자 선택 + 발송 페이지
3. 발송 이력 조회 페이지
4. 야간 발송 제한 로직

---

## 9. 참고사항

- **발송 인프라**: Microsoft Graph API (Azure AD OAuth) - 기존 시스템 활용
- **템플릿 엔진**: Thymeleaf 또는 자체 변수 치환 방식
- **모니터링**: 발송 실패 건수가 임계치 초과 시 관리자 알림 고려
