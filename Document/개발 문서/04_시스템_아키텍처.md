---

## 1. 개요

### 1.1 목적

본 문서는 BulC Homepage 시스템의 전체 아키텍처를 정의합니다.

### 1.2 아키텍처 원칙

| 원칙 | 설명 |
|------|------|
| **Stateless** | 서버에 세션 상태 저장하지 않음 (JWT 기반) |
| **계층 분리** | 프론트엔드/백엔드/데이터베이스 분리 |
| **컨테이너화** | Docker 기반 배포 |
| **RESTful** | REST API 설계 원칙 준수 |
