## 1. 프로젝트 개요

### 1.1 프로젝트 배경

화재 안전 시뮬레이션 및 교육 솔루션을 제공하는 통합 웹 플랫폼의 필요성에 따라, 사용자 인증, 제품 관리, 결제/구독, 라이선스 관리 기능을 갖춘 풀스택 웹 애플리케이션을 개발합니다.

### 1.2 프로젝트 목적

- METEOR, BUL:C, VR 등 화재 안전 관련 제품의 통합 소개 및 판매 플랫폼 구축
- 온라인 결제 및 구독 기반 서비스 모델 지원
- 소프트웨어 라이선스 발급 및 관리 시스템 구축
- 사용자 계정 관리 및 OAuth 소셜 로그인 지원

### 1.3 프로젝트 범위

| 구분 | 범위 |
|------|------|
| 대상 사용자 | 일반 사용자, 기업 고객, 관리자 |
| 제공 서비스 | 제품 소개, 온라인 결제, 구독 관리, 라이선스 관리 |
| 지원 플랫폼 | 웹 브라우저 (Chrome, Edge, Safari, Firefox) |
| 지원 결제 수단 | 카드, 계좌이체, 가상계좌, 간편결제 (토스페이먼츠) |

## 3. 주요 기능 요약

### 3.1 사용자 관리

| 기능 | 설명 |
|------|------|
| 회원가입 | 이메일 인증 기반 회원가입 |
| 소셜 로그인 | Google, Naver, Kakao OAuth2 |
| 로그인/로그아웃 | JWT 토큰 기반 인증 |
| 비밀번호 재설정 | 이메일 인증 코드 기반 |
| 계정 관리 | 정보 수정, 비밀번호 변경, 탈퇴 |

### 3.2 결제/구독

| 기능 | 설명 |
|------|------|
| 상품 조회 | 활성화된 상품 및 요금제 조회 |
| 결제 처리 | 토스페이먼츠 연동 결제 |
| 구독 관리 | 구독 조회, 자동 갱신 설정 |
| 빌링키 관리 | 자동 결제용 카드 등록/삭제 |
| 프로모션 | 쿠폰 검증 및 할인 적용 |

### 3.3 라이선스 관리

| 기능 | 설명 |
|------|------|
| 라이선스 발급 | 결제 완료 시 자동 발급 |
| 라이선스 검증 | 클라이언트 앱에서 실시간 검증 |
| 기기 활성화 | 기기별 세션 관리 (동시 접속 제한) |
| Heartbeat | 15분 주기 세션 갱신 |
| 오프라인 토큰 | 인터넷 미연결 시 로컬 검증 |

### 3.4 관리자 기능

| 기능 | 설명 |
|------|------|
| 사용자 관리 | 권한 변경, 계정 조회 |
| 상품 관리 | 상품 CRUD, 활성화/비활성화 |
| 요금제 관리 | 요금제 CRUD, 라이선스 플랜 연계 |
| 라이선스 관리 | 라이선스 조회, 활성화/정지 |

## 8. 용어 정의

| 용어 | 설명 |
|------|------|
| JWT | JSON Web Token, 인증 토큰 표준 |
| Access Token | API 인증용 단기 토큰 (1시간) |
| Refresh Token | 토큰 갱신용 장기 토큰 (7일) |
| 빌링키 | 자동 결제를 위한 카드 등록 키 |
| Heartbeat | 라이선스 세션 유지를 위한 주기적 통신 |
| Activation | 기기별 라이선스 활성화 세션 |
| OAuth | Open Authorization, 소셜 로그인 프로토콜 |
| Stateless | 서버에 세션 상태를 저장하지 않는 아키텍처 |
