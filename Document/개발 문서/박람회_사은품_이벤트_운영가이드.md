---

## 1. 이벤트 개요

| 항목 | 내용 |
|------|------|
| 행사명 | 2026 국제소방안전박람회 |
| 일정 | 2026년 5월 20일(수) ~ 5월 22일(금) |
| 장소 | 대구 EXCO · BulC 부스 |
| 사은품 지급 대상 | 본 홈페이지에 회원가입한 사용자 |
| 식별 방식 | 로그인한 사용자가 사은품 페이지를 부스 직원에게 직접 제시 |

회원가입을 완료한 사용자에게 사은품을 증정하는 행사로, **백엔드/DB 변경 없이 프론트엔드만으로 동작**하도록 구현되었습니다. 정적 설정 파일 1개로 모든 동작이 제어됩니다.

---

## 2. 동작 흐름

### 2.1 회원가입 → 사은품 페이지 자동 안내
1. 사용자가 회원가입 모달에서 가입 완료
2. 이벤트 활성 시 sessionStorage에 `pendingEventRedirect` 플래그 저장
3. 로그인 모달로 전환되며, 로그인 성공 시 `/event/booth-gift` 페이지로 자동 이동
4. OAuth 회원가입(`/oauth/setup-password`)의 경우 가입 직후 바로 페이지로 이동

### 2.2 마이페이지 상단 배너
- 이벤트 활성 기간 동안 마이페이지(`/mypage`) 상단에 배너 노출
- 배너 클릭 시 `/event/booth-gift` 로 이동
- 이벤트 비활성 시 배너는 자동으로 숨김

### 2.3 사은품 페이지 (`/event/booth-gift`)
- 로그인 필수 (비로그인 시 401 에러 페이지로 리다이렉트)
- 표시 항목: 사용자 이름 / 이메일 / 발급 시각 / 이벤트 기간 / 부스 위치
- 부스 직원이 페이지의 사용자 정보를 육안 확인 후 사은품 전달

## 3. 이벤트 On/Off 제어

**모든 제어는 단 하나의 파일로 수행됩니다:**

```
frontend/public/event-config.json
```

이 파일은 정적 파일로 서빙되므로 **수정 후 재배포만 하면 빌드 과정 없이 즉시 반영**됩니다 (브라우저 캐시는 최대 60초).

### 3.1 설정 항목

```json
{
  "boothGift": {
    "enabled": true,
    "startAt": "2026-05-20T00:00:00+09:00",
    "endAt": "2026-05-22T23:59:59+09:00",
    "title": "2026 국제소방안전박람회 사은품 이벤트",
    "subtitle": "회원가입을 완료해주셔서 감사합니다!",
    "description": "본 페이지를 부스에서 직원에게 보여주시면 사은품을 받으실 수 있습니다. 1인 1회에 한해 증정됩니다.",
    "boothInfo": "대구 EXCO · BulC 부스 (부스 위치 추후 안내)",
    "bannerTitle": "2026 국제소방안전박람회 사은품 이벤트 진행 중",
    "bannerCta": "사은품 페이지 보기",
    "bannerDescription": "대구 EXCO BulC 부스에서 페이지를 보여주시면 사은품을 증정합니다. (5/20 ~ 5/22)"
  }
}
```

| 필드 | 설명 | 비고 |
|------|------|------|
| `enabled` | 마스터 킬 스위치 | `false`면 기간과 무관하게 즉시 OFF |
| `startAt` | 이벤트 시작 시각 (ISO 8601, KST 권장) | 이 시각 이전엔 자동 비활성 |
| `endAt` | 이벤트 종료 시각 (ISO 8601, KST 권장) | 이 시각 이후엔 자동 비활성 |
| `title` | 사은품 페이지 메인 제목 | |
| `subtitle` | 사은품 페이지 부제 | |
| `description` | 사은품 페이지 본문 안내 | |
| `boothInfo` | 부스 위치 안내 | |
| `bannerTitle` | 마이페이지 배너 제목 | |
| `bannerCta` | 마이페이지 배너 버튼 텍스트 | |
| `bannerDescription` | 마이페이지 배너 설명 | |

### 3.2 활성화 조건

이벤트가 활성으로 판정되려면 **모든 조건이 충족**되어야 합니다:

1. `enabled === true`
2. 현재 시각이 `startAt ≤ now ≤ endAt` 범위 내
3. `startAt`, `endAt` 모두 유효한 ISO 8601 날짜 문자열

판정 로직: [`frontend/src/utils/eventConfig.ts`](../../frontend/src/utils/eventConfig.ts) 의 `isBoothGiftActive()`

## 4. 이벤트 OFF 시나리오별 절차

### 4.1 정상 종료 (이벤트 기간 만료 후)
**조치 불필요.** 시스템 시각이 `endAt`을 지나면 자동으로:
- 사은품 페이지: "이벤트가 진행되고 있지 않습니다" 표시 + 메인 이동 버튼
- 마이페이지 배너: 자동 숨김
- 회원가입 완료 시 일반 알림으로 변경 ("회원가입이 완료되었습니다. 로그인해주세요.")

### 4.2 긴급 중단 (이벤트 기간 중 즉시 OFF)
1. `frontend/public/event-config.json` 편집
2. `"enabled": true` → `"enabled": false` 로 변경
3. 변경된 파일을 배포 (Cloudflare Pages / nginx 정적 파일 서빙 환경 모두 동일)
4. 사용자 브라우저 측 캐시 TTL **최대 60초** 이후 모든 동작이 비활성화됨

> **참고:** `enabled: false`만으로도 즉시 OFF되므로, 날짜 필드는 그대로 유지해도 됩니다.

### 4.3 이벤트 일정 변경
1. `startAt` 또는 `endAt` 수정
2. 배포
3. **최대 60초 후** 새 날짜 기준으로 자동 활성/비활성 판정

### 4.4 이벤트 문구 변경
1. `title`, `subtitle`, `description`, `boothInfo`, `bannerTitle`, `bannerCta`, `bannerDescription` 등 자유롭게 수정
2. 배포
3. **최대 60초 후** 변경된 문구가 사용자에게 노출

### 4.5 행사 종료 후 코드 제거 (선택 사항)
이벤트가 완전히 끝난 뒤 추후 재사용 계획이 없다면 다음 파일들을 제거할 수 있습니다:

| 파일 | 비고 |
|------|------|
| `frontend/public/event-config.json` | 설정 파일 |
| `frontend/src/utils/eventConfig.ts` | 유틸리티 |
| `frontend/src/CategoryPages/Event/` | 사은품 페이지 디렉토리 전체 |
| `frontend/src/components/BoothGiftBanner.tsx` (+ CSS) | 배너 컴포넌트 |
| `frontend/src/index.tsx` | `/event/booth-gift` 라우트 제거 |
| `frontend/src/components/SignupModal.tsx` | `markPendingEventRedirect` 호출 부분 제거 |
| `frontend/src/components/LoginModal.tsx` | `consumePendingEventRedirect` 호출 부분 제거 |
| `frontend/src/pages/OAuthSetupPassword.tsx` | 이벤트 분기 제거 |
| `frontend/src/CategoryPages/MyPage/MyPage.tsx` | `<BoothGiftBanner />` 제거 |

> 단, 동일 인프라(`event-config.json`)로 향후 다른 박람회·행사를 진행할 수 있으므로 **유지하는 것을 권장**합니다.

## 5. 운영 시 주의사항

### 5.1 캐시 TTL 이해
- 브라우저 내부 캐시: **60초**
- nginx/CDN 캐시: 환경 설정에 따라 다름 (Cloudflare Pages 기본 정책 적용)
- 즉시 반영이 필요한 경우 사용자에게 "60초 후 새로고침" 안내 또는 CDN purge 수동 수행

### 5.2 부스 운영 체크리스트
- [ ] 부스 직원이 페이지의 **사용자 이름**과 **이메일**을 반드시 확인할 것
- [ ] 발급 시각(`발급 시각` 항목)을 통해 가짜 스크린샷 방지
- [ ] 동일 사용자 중복 수령 방지를 위해 별도 명단 수기 관리 권장 (시스템 추적 미지원)
- [ ] 사은품 페이지는 **로그인 사용자에게만** 보이므로, 부스에서 가입을 도울 경우 회원가입 후 로그인까지 진행 안내

### 5.3 비회원 회원가입 후 자동 안내
- 이메일 회원가입: 회원가입 완료 → 로그인 안내 → 로그인 성공 시 자동 이동
- OAuth 회원가입(구글/카카오/네이버): 비밀번호 설정 완료 → 자동 이동
- 두 경로 모두 `sessionStorage` / 함수 분기로 처리되므로 별도 안내 불필요

### 5.4 백엔드/DB 영향 없음
본 기능은 **순수 프론트엔드 정적 자원**으로 구성되며 다음 항목과 무관:
- 백엔드 코드 / API
- 데이터베이스 스키마 / 데이터
- 인증/세션 메커니즘 (기존 JWT 그대로 사용)
- 환경 변수
