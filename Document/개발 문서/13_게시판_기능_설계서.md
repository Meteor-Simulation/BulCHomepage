# 13. 게시판 기능 설계서

> 작성일: 2026-04-20
> 기술 스택: React 19 + Spring Boot 3.2 + PostgreSQL 16 (기존 프로젝트 통합)

---

## 1. 개요

회원이 글과 이미지를 작성할 수 있는 게시판 시스템.
비회원/회원 등급에 따라 글 열람 범위가 달라지며, 새로고침만으로 최신 글이 반영됩니다.

---

## 2. 핵심 요구사항

### 기능
- 게시글 작성 / 수정 / 삭제 (회원 전용)
- Rich Text Editor (번호 목록, 불릿, 볼드, 이탤릭 등)
- 이미지 업로드 (본문 내 삽입)
- 이미지 위 도형/텍스트 주석 기능
- 새로고침 시 최신 게시글 반영 (서버 재시작 불필요)

### 접근 권한

| 등급 | 글 목록 | PUBLIC 열람 | MEMBER 열람 | STAFF 열람 | 글 작성 | 본인 글 수정/삭제 | 모든 글 관리 |
|------|---------|------------|------------|-----------|---------|-----------------|-------------|
| 비회원 (guest) | O | O | X | X | X | X | X |
| 회원 (user) | O | O | O | X | O | O | X |
| 스태프 (manager/admin) | O | O | O | O | O | O | O |

- 글 목록은 **모든 등급에서 볼 수 있음** (제목, 작성자, 날짜 등)
- 권한이 없는 글을 클릭하면 본문 대신 안내 메시지 표시
- 작성자가 글 작성 시 공개 범위를 선택
- 게시판에서 **manager와 admin은 동일 권한** (스태프로 통합)

### 글 공개 범위

| 값 | 최소 열람 등급 | 설명 |
|---|-------------|------|
| `PUBLIC` | 비회원 | 누구나 열람 가능 |
| `MEMBER` | 회원 (user) | 로그인한 회원부터 열람 가능 |
| `STAFF` | 스태프 (manager/admin) | 매니저/관리자만 열람 가능 (내부 공지 등) |

> 등급 계층: staff(admin+manager) > user > guest

## 4. DB 테이블 설계

### 4.1 posts (게시글)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK (IDENTITY) | 게시글 ID |
| author_id | UUID FK → users.id | 작성자 |
| title | VARCHAR(200) | 제목 |
| content_html | TEXT | 본문 (HTML) |
| visibility | VARCHAR(20) | `PUBLIC` / `MEMBER` / `STAFF` |
| view_count | INT | 조회수 |
| is_deleted | BOOLEAN | 소프트 삭제 |
| created_at | TIMESTAMP | 작성일 |
| updated_at | TIMESTAMP | 수정일 |

### 4.2 post_images (게시글 이미지)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK (IDENTITY) | 이미지 ID |
| post_id | BIGINT FK → posts.id | 게시글 ID |
| image_url | VARCHAR(500) | 이미지 경로 (상대 or S3 URL) |
| original_name | VARCHAR(255) | 원본 파일명 |
| file_size | BIGINT | 파일 크기 (bytes) |
| created_at | TIMESTAMP | 업로드일 |

### 4.3 annotations (이미지 주석)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK (IDENTITY) | 주석 ID |
| image_id | BIGINT FK → post_images.id | 이미지 ID |
| type | VARCHAR(20) | `CIRCLE` / `RECT` / `TEXT` |
| x | DOUBLE | X 좌표 |
| y | DOUBLE | Y 좌표 |
| width | DOUBLE | 너비 (RECT) |
| height | DOUBLE | 높이 (RECT) |
| radius | DOUBLE | 반지름 (CIRCLE) |
| text | VARCHAR(500) | 텍스트 내용 (TEXT 타입) |
| color | VARCHAR(20) | 색상 |
| created_at | TIMESTAMP | 생성일 |

## 5. API 설계

### 5.1 게시글

| 메서드 | 엔드포인트 | 인증 | 설명 |
|--------|-----------|------|------|
| GET | `/api/v1/posts` | 불필요 | 게시글 목록 (페이징, 검색) |
| GET | `/api/v1/posts/{id}` | 조건부 | 게시글 상세 (MEMBERS_ONLY는 인증 필요) |
| POST | `/api/v1/posts` | 필수 | 게시글 작성 |
| PUT | `/api/v1/posts/{id}` | 필수 | 게시글 수정 (본인 or 관리자) |
| DELETE | `/api/v1/posts/{id}` | 필수 | 게시글 삭제 (본인 or 관리자) |

#### 목록 조회 쿼리 파라미터

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| page | int | 페이지 번호 (기본 0) |
| size | int | 페이지 크기 (기본 20) |
| search | string | 제목/본문 검색어 |
| sort | string | 정렬 기준 (기본 `createdAt,desc`) |

#### 목록 응답 예시

```json
{
  "content": [
    {
      "id": 1,
      "title": "제목",
      "authorName": "작성자",
      "visibility": "PUBLIC",
      "viewCount": 42,
      "createdAt": "2026-04-20T10:00:00",
      "thumbnailUrl": "/uploads/posts/thumb_xxx.jpg"
    }
  ],
  "totalPages": 5,
  "totalElements": 98,
  "number": 0
}
```

#### 상세 응답 — 접근 권한 로직

```
요청자 등급 < 글의 visibility 등급 → 200 OK (제목/작성자만 반환, 본문 제외, restricted: true)
요청자 등급 >= 글의 visibility 등급 → 200 OK (전체 반환, restricted: false)

예시:
  비회원 → PUBLIC 글     → 열람 가능
  비회원 → MEMBER 글     → 제한 ("회원에게만 공개된 게시글입니다")
  회원   → MEMBER 글     → 열람 가능
  회원   → STAFF 글      → 제한 ("스태프에게만 공개된 게시글입니다")
  스태프 → STAFF 글      → 열람 가능
```

### 5.2 이미지 업로드

| 메서드 | 엔드포인트 | 인증 | 설명 |
|--------|-----------|------|------|
| POST | `/api/v1/posts/images` | 필수 | 이미지 업로드 (multipart) |

#### 업로드 제한
- 최대 파일 크기: 5MB
- 허용 확장자: jpg, jpeg, png, gif, webp
- 파일명: UUID로 변환 저장

#### 응답

```json
{
  "imageId": 1,
  "imageUrl": "/uploads/posts/abc123.jpg"
}
```

### 5.3 이미지 주석

| 메서드 | 엔드포인트 | 인증 | 설명 |
|--------|-----------|------|------|
| GET | `/api/v1/posts/images/{imageId}/annotations` | 조건부 | 주석 목록 |
| POST | `/api/v1/posts/images/{imageId}/annotations` | 필수 | 주석 저장 (일괄) |

#### 주석 저장 요청 예시

```json
{
  "annotations": [
    { "type": "CIRCLE", "x": 120, "y": 200, "radius": 50, "color": "#ff0000" },
    { "type": "RECT", "x": 50, "y": 100, "width": 200, "height": 150, "color": "#0000ff" },
    { "type": "TEXT", "x": 80, "y": 300, "text": "문제 영역", "color": "#333333" }
  ]
}
```

### 6.4 등급별 UI 차이

| 요소 | 비회원 | 회원 | 스태프 (manager/admin) |
|------|--------|------|----------------------|
| 글 목록 | 전체 표시 | 전체 표시 | 전체 표시 |
| 잠금 아이콘 | MEMBER/STAFF 글 | STAFF 글 | 없음 |
| 권한 없는 글 클릭 | 안내 메시지 + 로그인 유도 | 안내 메시지 | — |
| 글쓰기 버튼 | 숨김 | 표시 | 표시 |
| 공개 범위 선택 | — | PUBLIC, MEMBER | PUBLIC, MEMBER, STAFF |
| 수정/삭제 | — | 본인 글만 | 모든 글 |

## 7. 파일 저장 전략

### 개발 환경
- 로컬 디렉토리: `backend/uploads/posts/`
- URL 매핑: `/uploads/posts/xxx.jpg`

### 운영 환경
- Docker volume 또는 S3
- nginx에서 `/uploads/` 경로를 volume으로 서빙

### 보안
- 파일명 UUID 변환 (원본 파일명 노출 방지)
- 확장자 화이트리스트 검증 (서버 측)
- 파일 Content-Type 검증 (매직 바이트)
- 최대 5MB 제한

## 10. 참고사항

- 기존 프로젝트의 인증 시스템(JWT HttpOnly 쿠키)을 그대로 활용
- 기존 RBAC(admin/manager/user) 역할 체계와 통합
- `is_deleted` 소프트 삭제로 데이터 보존
- 게시글 본문은 HTML로 저장 (TipTap 출력), XSS 방지를 위해 서버 측 sanitize 필수
