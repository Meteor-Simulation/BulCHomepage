## 1. 환경 비교

| 항목 | 로컬 (개발) | 운영 (Production) |
|------|-----------|------------------|
| **프론트엔드** | `npm start` (localhost:3000) | Cloudflare Pages (자동 배포) |
| **백엔드** | `./gradlew bootRun` (localhost:8080) | Docker (`docker compose -f docker-compose.prod.yml`) |
| **DB** | Docker 또는 로컬 PostgreSQL | Docker (`bulc-db-prod`) |
| **DB 사용자** | `postgres` / `postgres123` | `bulc_prod_user` / (별도 비밀번호) |
| **DB 포트** | 5432 (또는 5433 테스트용) | 5432 (내부 네트워크, 외부 미노출) |
| **Spring Profile** | `dev` | `prod` |
| **YAML 검증** | DevTools 관대 처리 | **엄격 (중복 키 → 시작 실패)** |
| **쿠키 Secure** | `false` (HTTP 허용) | `true` 변경 필요 (HTTPS) |

## 2. application.yml 수정 시 주의사항

### 중복 키 금지

```yaml
# 잘못된 예 — server: 가 두 번
server:
  port: 8080

# ... 중간에 다른 설정 ...

server:                    # ← DuplicateKeyException 발생!
  tomcat:
    max-http-form-post-size: 1MB
```

```yaml
# 올바른 예 — 하나의 server: 블록에 통합
server:
  port: 8080
  tomcat:
    max-http-form-post-size: 1MB
```

### 수정 전 체크 방법

```bash
# 중복 최상위 키 확인
grep -n "^[a-z]" application.yml | sort -t: -k2 | uniq -d -f1
```

### 로컬 vs 운영 차이

- 로컬: Spring DevTools가 중복 키를 **무시하거나 마지막 값으로 덮어씀**
- 운영: SnakeYAML이 **DuplicateKeyException을 던져서 시작 실패**
- **로컬에서 문제 없다고 운영에서도 안전하지 않음**

## 3. 배포 절차

### 백엔드 (Oracle Cloud Docker)

```bash
# 1. 서버 접속
ssh -i ~/.ssh/oracle_homepage.pem ubuntu@168.107.19.139

# 2. 코드 업데이트
cd ~/BulCHomepage && git pull origin main

# 3. DB 마이그레이션 (필요 시)
docker compose -f docker-compose.prod.yml exec -T database \
  sh -c 'psql -U bulc_prod_user -d bulc_homepage_db' < database/migrations/V날짜__이름.sql

# 4. 빌드 & 재시작
docker compose -f docker-compose.prod.yml up -d --build backend

# 5. 상태 확인
docker logs bulc-backend-prod --tail 20
curl -s http://localhost:8080/api/health
```

### 프론트엔드 (Cloudflare Pages)

main 브랜치 push 시 자동 배포 (별도 작업 불필요)
