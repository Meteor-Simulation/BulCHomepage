# 12. 보안 체크리스트

> 최종 점검일: 2026-04-20
> 대상: BulC Homepage (Spring Boot 3.2 + React 19 + PostgreSQL 16)

## 1. 인증 (Authentication)

| 항목 | 중요도 | 상태 | 비고 |
|------|--------|------|------|
| JWT (Access + Refresh Token) | 상 | **구현됨** | Access 10분, Refresh 7일, HS256, JJWT 0.12.3 |
| Refresh Token Rotation (RTR) | 상 | **구현됨** | DB 저장, 토큰 탈취 감지 시 전체 세션 무효화 |
| 비밀번호 해싱 (BCrypt) | 상 | **구현됨** | `BCryptPasswordEncoder`, SecurityConfig |
| 로그인 시도 제한 | 상 | **부분 구현** | 이메일 인증에만 적용 (5회/60분, 초과 시 24시간 잠금). **로그인 엔드포인트에는 미적용** |
| 세션 관리 | 중 | **구현됨** | 단일 디바이스 정책 (사용자당 Refresh Token 1개), 토큰 블랙리스트 |
| OAuth 2.0 소셜 로그인 | 중 | **구현됨** | Naver, Kakao, Google 연동 완료 |
| 2단계 인증 (2FA) | 중 | **미구현** | OTP/SMS 인증 없음 |
| 비밀번호 정책 | 중 | **구현됨** | 최소 8자, 영문+숫자+특수문자 필수 (ValidationConfig) |
| 계정 잠금 정책 | 중 | **미구현** | 로그인 N회 실패 시 잠금 기능 없음 |

### JWT 구현 상세

| 항목 | 상태 | 현재값 |
|------|------|--------|
| Access Token 만료시간 | **구현됨** | 10분 (600,000ms) |
| Refresh Token 만료시간 | **구현됨** | 7일 (604,800,000ms) |
| Secret Key 256bit 이상 | **구현됨** | 384bit 키 사용 |
| 환경변수 관리 | **구현됨** | `.env`에서 주입 |
| 토큰 블랙리스트 | **구현됨** | 로그아웃 시 Access Token 무효화 |
| HttpOnly 쿠키 저장 | **부분** | 백엔드에서 HttpOnly 쿠키 설정하지만, **프론트엔드가 localStorage에도 저장** (보안 취약) |

## 4. 입력값 검증 및 공격 방지

| 항목 | 중요도 | 상태 | 비고 |
|------|--------|------|------|
| SQL Injection 방지 | 상 | **구현됨** | Spring Data JPA + QueryDSL, Raw SQL 없음 |
| XSS 방지 | 상 | **구현됨** | HttpOnly 쿠키, HTML 이스케이프 (ContactController), CSP 헤더 |
| CSRF 방지 | 상 | **구현됨** | Stateless JWT이므로 CSRF 비활성화 (적절한 설계) |
| 입력값 검증 (서버) | 상 | **구현됨** | `@Valid`, `ValidationConfig`, `PasswordValidator` |
| 파일 업로드 검증 | 상 | **미구현** | 파일 업로드 엔드포인트 없음 (현재 불필요) |
| Rate Limiting | 중 | **부분 구현** | 이메일 인증 (5회/60분), 리딤 코드 (5회/60초). **전역 API Rate Limit 없음** |
| 요청 크기 제한 | 중 | **미설정** | `application.yml`에 `max-http-post-size` 미설정 |

### CSP 헤더 상세 (nginx.conf)

```
default-src 'self';
script-src 'self' 'unsafe-inline' js.tosspayments.com;
style-src 'self' 'unsafe-inline' fonts.googleapis.com;
img-src 'self' data: blob:;
font-src 'self' fonts.gstatic.com;
frame-src js.tosspayments.com *.tosspayments.com;
connect-src 'self' api.tosspayments.com graph.microsoft.com;
```

## 5. 결제 시스템 보안

| 항목 | 중요도 | 상태 | 비고 |
|------|--------|------|------|
| PG사 연동 (직접 카드처리 금지) | 상 | **구현됨** | Toss Payments API 사용, 카드 정보 미접촉 |
| 서버 간 결제 검증 | 상 | **구현됨** | Toss API에 재검증 요청 (PaymentWebhookController) |
| 결제 금액 서버 재검증 | 상 | **구현됨** | DB의 PricePlan 금액과 비교 (PaymentService) |
| 거래 로그 기록 | 상 | **구현됨** | Payment 엔티티 + ActivityLog에 모든 시도 기록 |
| Webhook 검증 | 상 | **부분 구현** | Toss API 재조회로 검증하지만, **HMAC 서명 검증 미구현** |
| 멱등성 키 | 중 | **구현됨** | `orderId` 유니크 제약으로 중복 결제 방지 |
| PCI-DSS 준수 | 중 | **해당 없음** | 카드 정보 미처리 (Toss에 위임) |

### 결제 플로우 검증

```
1. 클라이언트: 상품 선택 → 서버: 주문 생성, DB 기준 금액 확정  ✅
2. 클라이언트: Toss 결제창 호출                                ✅
3. Toss: 결제 처리                                            ✅
4. 서버: 결제 승인 API 호출 + 금액 재검증                       ✅
5. 서버: Webhook 수신 → Toss API 재조회로 상태 확인             ✅
6. 서버: @Transactional로 구독+라이선스 발급 원자적 처리         ✅
```

## 조치 필요 항목 (우선순위별)

### 긴급 (Critical)

| # | 항목 | 현황 | 조치 |
|---|------|------|------|
| 1 | localStorage 토큰 저장 | AuthContext.tsx에서 accessToken, refreshToken을 localStorage에 저장 | HttpOnly 쿠키 전용으로 전환, localStorage 사용 제거 |

### 높음 (High)

| # | 항목 | 현황 | 조치 |
|---|------|------|------|
| 2 | 로그인 시도 제한 | `/api/auth/login`에 Rate Limit 없음 | IP 기반 로그인 시도 제한 추가 (5회/15분) |
| 3 | 계정 잠금 정책 | 로그인 실패 시 잠금 없음 | N회 실패 시 일시 잠금 구현 |
| 4 | PII 암호화 | 이메일/전화번호 평문 저장 | AES-256 암호화 적용 |
| 5 | Webhook 서명 검증 | Toss API 재조회만 수행 | HMAC-SHA256 서명 검증 추가 |
| 6 | CORS OAuth `*` 허용 | OAuth 엔드포인트에서 전체 origin 허용 | 데스크톱 앱 origin만 허용하도록 제한 |

### 중간 (Medium)

| # | 항목 | 현황 | 조치 |
|---|------|------|------|
| 7 | 전역 Rate Limiting | 특정 엔드포인트만 적용 | Bucket4j 또는 nginx rate_limit 모듈 적용 |
| 8 | 요청 크기 제한 | 미설정 | `server.tomcat.max-http-post-size: 1MB` 설정 |
| 9 | 2FA | 미구현 | TOTP (Google Authenticator) 지원 추가 |
| 10 | 중앙 로깅/알림 | stdout만 | ELK 또는 CloudWatch 연동 |
| 11 | DB 접근 계정 분리 | 단일 postgres 계정 | 읽기 전용 / 쓰기 전용 분리 |
| 12 | API 버전 일관성 | `/api/v1/`과 `/api/` 혼재 | 전체 `/api/v1/`로 통일 |

### 낮음 (Low)

| # | 항목 | 현황 | 조치 |
|---|------|------|------|
| 13 | SRI (외부 스크립트) | 미적용 | Toss, Google Fonts에 integrity 속성 추가 |
| 14 | SameSite 쿠키 | 개발환경 Lax | 운영환경에서 Strict로 변경 |
